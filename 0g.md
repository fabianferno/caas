## 0G Features Used

### 1. 0G Chain -- ERC-7857 Intelligent NFTs

Every agent is minted as an INFT on 0G Chain. The contract supports:

- **`mint()`** -- Mint agent with encrypted IntelligentData array (soul, skills, config hashes)
- **`iTransfer()`** -- Secure transfer with re-encryption for new owner (personality + memory transfer with ownership)
- **`iClone()`** -- Clone agent templates (marketplace / agent forking)
- **`authorizeUsage()`** -- Grant interaction rights without ownership transfer
- **`revokeAuthorization()`** -- Revoke interaction rights
- **`delegateAccess()`** -- Delegate to TEE oracle for access control
- **`intelligentDataOf()`** -- Read encrypted data hashes for any token

**Why INFT:** Agents are tradeable digital assets. Unlike regular NFTs, ERC-7857 INFTs support encrypted metadata that transfers securely with ownership. An agent's personality, skills, and memory are cryptographically bound to the token.

**Files:**
- `contracts/AgentNFT.sol` -- Full ERC-7857 UUPS implementation (171 lines)
- `contracts/interfaces/IERC7857.sol` -- ERC-7857 interface
- `contracts/interfaces/IERC7857DataVerifier.sol` -- AccessProof, OwnershipProof, TransferValidityProof structs
- `contracts/interfaces/IERC7857Metadata.sol` -- Metadata interface
- `contracts/scripts/deploy.ts` -- UUPS proxy deployment
- `contracts/scripts/demo-mint.ts` -- Demo script (deploy + mint 3 agents + verify on-chain)

**SDKs:** Hardhat v2.22, OpenZeppelin Upgradeable v5, @openzeppelin/hardhat-upgrades v3

### 2. 0G Storage -- Encrypted Agent Memory & Soul

Agent data is encrypted with AES-256-GCM and uploaded to 0G Storage. ENS text records point to the content hashes.

**What gets stored:**
- `soul.md` -- Agent system prompt and personality rules
- `skills.json` -- OpenAPI skill specifications the agent can call
- `config.json` -- Agent configuration (tone, channels, guardrails)
- `memory/` -- Conversation logs and learned preferences

**Encryption flow:**
1. Generate AES-256-GCM key
2. Encrypt each blob (IV + ciphertext + authTag)
3. Upload via `@0gfoundation/0g-ts-sdk` Indexer
4. Get back rootHash per blob
5. Compute merkle root over all hashes
6. Store merkle root in ENS text record (`caas.storage`)
7. Store individual hashes as IntelligentData in the INFT on 0G Chain

**Why 0G Storage:** Agent personality is verifiable and tamper-evident. Anyone can check that an agent's soul hasn't been silently altered -- modifications create new hashes. Decentralized storage means no single point of failure for agent identity.

**Files:**
- `miniapp/src/lib/inft-uploader.ts` -- AES-256-GCM encryption + 0G Storage upload (83 lines)
- `agent/src/memory/zerog.ts` -- 0G Storage memory backend for runtime (93 lines)
- `miniapp/src/app/api/agent/deploy/route.ts` -- Orchestrates upload during deployment (lines 70-85)

**SDKs:** `@0gfoundation/0g-ts-sdk` v1.0
**Indexer:** https://indexer-storage-standard.0g.ai
**RPC:** https://evmrpc-testnet.0g.ai

### 3. 0G Compute -- Decentralized LLM Inference

0G Compute is the primary LLM provider for all agents. The runtime discovers available chatbot services, manages ledger/funds, and routes inference requests through the 0G Compute Network.

**How it works:**
1. Create broker via `createZGComputeNetworkBroker(wallet)`
2. List available services, filter for chatbot type
3. Create ledger and deposit funds
4. Transfer funds to chosen provider
5. Acknowledge provider signer
6. Send OpenAI-compatible chat completions through `{endpoint}/v1/proxy/chat/completions`
7. Include signed request headers from `broker.inference.getRequestHeaders()`

**Fallback:** If 0G Compute is unavailable, automatically falls back to AWS Bedrock (Claude/Nova). The `FallbackLLMProvider` class handles this transparently.

**Why 0G Compute:** TEE-backed decentralized inference means verifiable AI outputs. Unlike centralized APIs, agents can prove their responses were generated by a specific model without tampering.

**Files:**
- `agent/src/core/llm.ts` -- `ZeroGLLMProvider` (lines 188-334), `FallbackLLMProvider` (lines 338-377)
- `agent/src/index.ts` -- Initialization (lines 35-42)

**SDKs:** `@0glabs/0g-serving-broker` v0.7.4

## How the 0G Stack Fits Together

```
User creates agent in World Mini App
    |
    v
[0G Storage] Encrypt & upload soul.md, skills.json, config.json
    |         Returns content hashes + merkle root
    v
[0G Chain]  Mint ERC-7857 INFT with content hashes as IntelligentData
    |         Returns tokenId + tx hash
    v
[ENS]       Write caas.inft (tokenId) + caas.storage (merkle root)
    |
    v
Agent container starts
    |
    v
[0G Compute] Agent uses decentralized inference for all LLM calls
[0G Storage] Agent persists conversation memory
[0G Chain]   INFT tracks ownership, authorization, cloning
```

All three 0G layers are used in the critical path:
- **Storage** at deployment (encrypt + upload) and runtime (memory persistence)
- **Compute** at runtime (every LLM inference call)
- **Chain** at deployment (INFT mint) and for ongoing ownership/composability

## Key Differentiators for 0G Judges

1. **Full-stack 0G integration** -- Not just one layer. We use Storage (encrypted agent data), Compute (decentralized inference), and Chain (ERC-7857 INFTs) together in a single deployment pipeline.

2. **First production use of ERC-7857 on 0G Chain** -- Agents as Intelligent NFTs with encrypted, transferable personality and memory. Supports iTransfer, iClone, authorizeUsage.

3. **Verifiable agent personality** -- Soul stored on 0G Storage with content hashes on-chain. Anyone can verify an agent's personality hasn't been tampered with.

4. **Agent composability via INFTs** -- Clone agents (iClone), authorize third-party usage (authorizeUsage), transfer ownership with re-encryption (iTransfer). Enables an agent marketplace.

5. **Persistent encrypted memory** -- Conversation history and learned preferences stored on 0G Storage, not a central database. Memory survives container restarts and ownership transfers.

6. **TEE-backed inference** -- 0G Compute provides verifiable AI outputs. Combined with World ID (human verification), agents have both verifiable intelligence and verifiable humanity.
